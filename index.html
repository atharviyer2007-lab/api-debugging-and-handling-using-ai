Workflow — How to Use
1
Pick a failure scenario
Click the scenario selector in the toolbar — choose Deployment Regression, DB Overload, or Memory Leak. Each scenario simulates a distinct real-world failure pattern with its own log signatures and service impact.
2
Inject the failure
Click ⚡ Simulate. Services degrade, error rates spike, incident timeline activates. Compare against the SLA baseline in the charts.
3
Run AI root-cause analysis
Click 🧠 Analyze. The agent sequentially calls 5 MCP-style tools, correlates signals, and returns a root cause, technical explanation, and remediation plan.
4
Toggle dark / light mode
Use the ☀ / ◐ button in the top-right. Preferences are saved to localStorage across sessions.
MCP-Style Agent Tools
get_logs()
Collects recent log entries (INFO/WARN/ERROR/DEBUG/CRITICAL) from the active log pool. Pool switches on failure injection.
get_metrics()
Fetches live metrics — avg latency, error rate, uptime, req/min, per-service health. Also appends to rolling 60-point history.
get_deployments()
Returns last 5 deployments: service, version, commit hash, author, status, environment.
detect_anomaly()
Analyses metrics + logs. Fires when error_rate > threshold OR latency > SLA. Returns signals and pattern fingerprint.
analyze_incident()
Correlates anomaly with scenario-specific root cause knowledge. Generates root cause, technical explanation, severity, and step-by-step fix.
get_history() NEW
Returns rolling 60-point metrics history for SLA comparison overlays on the charts.

Backend API Endpoints
GET /status
Service status, active incident ID, incident type, and configured thresholds.
GET /metrics/history
Rolling 60-point metrics history for SLA baseline comparison.
GET /alerts
All fired threshold alerts with type, value, and timestamp.
GET /scenarios
Available failure scenarios with key, label, and description.
POST /simulate-failure
Body: {"scenario": "deployment-regression"}. Activates incident with scenario-specific logs and metrics.
POST /analyze
Runs 5-tool MCP pipeline. Returns root_cause, severity, explanation, suggested_fix, confidence, scenario.
